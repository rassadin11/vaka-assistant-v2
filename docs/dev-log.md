# Журнал разработки

Делаем Telegram-бота — личного помощника, которому можно написать или наговорить голосом: посчитай траты, напомни о встрече, глянь календарь и почту в Google, найди что-то в интернете, ответь по вопросу из загруженного PDF-файла. Внутри бота работает языковая модель (ИИ, похожий на ChatGPT), которая понимает обычный текст и сама решает, какую функцию бота вызвать — записать трату, поставить напоминание и так далее. Сервис будет платным по подписке, ориентировочно 249 ₽ в месяц.

## 9 июля 2026

Сегодня в основном закладывали фундамент — того, что видно глазами пользователя, пока нет, но без этого дальше двигаться нельзя.

**Документ требований.** Написали и зафиксировали, что бот должен уметь, как он будет зарабатывать (подписка) и как обращаться с данными пользователей — что можно, что нельзя, какие есть обещания по приватности. Это теперь опорный документ: любое решение по проекту сверяется с ним.

**Скелет проекта на GitHub.** Завели репозиторий — место, где хранится весь код проекта с историей изменений. Настроили так, что при каждом изменении кода запускается автоматическая проверка: код смотрят на предмет ошибок, прогоняют тесты (проверки «работает ли то, что должно работать») и отдельно ищут, не попал ли случайно в код пароль или ключ доступа. Первый прогон этих проверок — весь зелёный, то есть без замечаний.

**Регистрации.** Зарегистрировали бота в Telegram (точнее, двух — один для тестов, второй будущий боевой) и получили доступ к сервису, через который бот обращается к ИИ-модели. Все ключи доступа убрали из кода в отдельное защищённое хранилище секретов, чтобы они не могли случайно утечь вместе с кодом.

**Мини-облако для разработки.** Подняли локальное окружение для разработки: одной командой поднимаются база данных, где будут храниться все данные пользователей, очереди сообщений (механизм, который передаёт задачи между частями бота) и хранилище для паролей и ключей. Это уменьшенная копия того, что потом будет работать на боевом сервере — удобно проверять всё локально, не трогая ничего боевого.

**Изоляция данных пользователей — в работе.** Начали делать так, чтобы один пользователь физически не мог получить доступ к данным другого, даже если в коде где-то ошибка. Это делается на уровне самой базы данных — правило, которое действует всегда, независимо от того, что происходит в коде бота. Пока не закончено, доделываем.

**Организационный момент.** Написание кода теперь по большей части делегировано ИИ-агенту (модель GPT-5.5, через инструмент Codex) — он получает чёткое задание и пишет код, а я (Claude) ставлю ему задачи, проверяю результат тестами и ручной проверкой и принимаю или отправляю на доработку. Так дешевле и не медленнее, но ответственность за качество всё равно на мне.
