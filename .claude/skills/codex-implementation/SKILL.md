# Codex Implementation — реализация по спеке

## Предусловия (все обязательны)

1. Явная просьба или подтверждение пользователя на запуск с правкой файлов.
2. Чёткая спека: целевой `plan/stage-N.md` с DoD либо самодостаточное ТЗ. Если в плане нет ответа на возникший вопрос — остановиться и дописать план (регламент CLAUDE.md), а не отдавать неопределённость Codex.
3. `codex --version` работает; иначе — установка `npm install -g @openai/codex` + `codex login`.

## 1. Составление промпта

Codex не видит наш диалог и не знает регламент проекта. Промпт самодостаточный:

- задача + критерии готовности (DoD из stage-N.md), абсолютные пути к спеке и целевым файлам;
- инварианты кода из `CLAUDE.md`, релевантные задаче: RLS (`SET LOCAL app.user_id`), идемпотентность mutating-инструментов, `user_id` только из TaskContext, тесты без сети (MockLLMProvider/MockSTTProvider), LLM только через LLMProvider, timestamptz в UTC;
- конвенции: код/идентификаторы/docstrings — английский; тексты для пользователей — русский;
- что НЕ трогать: плановые документы (`plan/`, `implementation_plan_v1.md`, `tools_registry_v1.md`, `CLAUDE.md`), конфиги секретов;
- что вернуть: список изменённых файлов + краткое обоснование решений.

Секреты в промпт не включать.

## 2. Запуск (с правкой файлов)

```
codex exec --full-auto --skip-git-repo-check --cd "D:\claude-projects\personal-assistant" "<промпт>"
```

- Явный `timeout` (до 600000 мс) или фоновый запуск с ожиданием файла-отчёта — реализация почти всегда дольше 2 минут.
- Параллельные запуски с правками — только с изоляцией worktree; требует git-репозиторий, до `git init` параллельные mutating-запуски запрещены (последовательные — можно).
- Длинный промпт — через файл в scratchpad: `codex exec ... "$(cat <файл>)"`.

## 3. Приёмка диффа (обязательна, силами Claude)

1. Прочитать дифф целиком — каждый изменённый файл.
2. Проверить инварианты CLAUDE.md и соответствие DoD этапа.
3. `pytest` зелёный; линтеры (ruff, mypy), если уже настроены.
4. Секретов в диффе нет (gitleaks, когда настроен).
5. Провал приёмки: мелочи доправить силами Claude; системные ошибки — перезапустить Codex с уточнённым промптом или эскалировать на более умную модель по таблице (эскалация без вопросов — дефолт, не лимит).
6. Отметить прогресс в `plan/progress.md` только после полной приёмки.

## 4. Итог пользователю

Что делегировано, какие файлы изменены, результаты приёмки (тесты/инварианты), спорные решения Codex и моя оценка.
