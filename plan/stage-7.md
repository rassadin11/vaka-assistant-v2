# Этап 7. Онбординг и биллинг (5–10 дней)

Часть мастер-плана: [../implementation_plan_v1.md](../implementation_plan_v1.md) — там архитектурные решения, сводный график и регламент передачи задач.

7.1. Tribute: webhook-endpoint (верификация подписи) → users.status, paid_until; события оплаты/отмены/неуспешного списания. [Обсудить: маппинг событий Tribute на статусы]
7.2. Триал: 7 дней ИЛИ 50 сообщений — что наступит раньше; без карты; один триал на tg user_id. Внутри триала действуют полные лимиты тарифа v1 (отдельной конфигурации лимитов нет). Воронка /start → триал → оплата с метриками конверсии.
7.3. Grace-период 48 ч после неуспешного списания; уведомление за 3 дня до окончания подписки; ежедневная задача деактивации просроченных.
7.4. Тариф v1: 249₽/мес, лимиты из этапа 5; поле users.plan и множители бюджета — задел под Pro-тариф с выбором моделей. [Обсудить: годовая подписка; запасной канал оплаты (Stars) для пользователей вне зоны Tribute; конкретные лимиты и юнит-экономика — отложенное обсуждение №11]
7.5. GDPR: команда экспорта данных (архив в чат), команда полного удаления (каскад по всем таблицам + vault + S3), автоудаление данных неактивных отменённых подписок через N дней. [Обсудить: сроки хранения]
7.6. Публичная оферта (черновик из 0.6) финализируется и публикуется; ссылка в онбординге до оплаты.
7.7. Отключение ручного одобрения из 2.9; допуск по оплате/триалу.

[DoD] Полный цикл на тестовом платеже: триал → оплата → активация → отмена → grace → деактивация → удаление данных.
