# Персона ассистента, итерация 2: грамматический род

Статус: решение владельца 2026-07-18 («оформи как следующую задачу»). Очередь: строго после полной приёмки и деплоя итерации 1 (plan/assistant-persona-iteration-1.md) — надстройка над её механикой.

## 1. Цель

Пользователь может задать род ассистента (женский/мужской/нейтральный): ассистент говорит о себе в правильных грамматических формах русского языка («записала», «готова», «нашла») во всех ответах.

## 2. Скоуп

1. Поле `gender` в `users.assistant_profile` (jsonb, миграция не нужна): `"female" | "male" | "neutral"`, отсутствие = текущее поведение (нейтрально-мужские формы по умолчанию модели). Род задаётся ТОЛЬКО явным полем — не выводить из имени (имя «Алиса» без поля gender род не меняет; модель может подстроиться сама — не мешаем, но не форсируем).
2. `set_assistant_persona`: аргумент `gender` (Literal, та же семантика частичного обновления, что и остальные поля); `clear_assistant_persona` сбрасывает вместе со всем профилем. Правки реестра §7a — этой спекой (args дополняются полем gender).
3. Блок C2: при заполненном gender — строка вида `- Grammatical gender for self-reference in Russian: feminine (готова, записала, нашла)` (формы-примеры прямо в строке — усиливают устойчивость). Бюджет C2 остаётся ≤200 токенов.
4. Промпт не трогаем (рамка 16a–16c уже покрывает: род — часть косметики персоны). Если eval покажет нестабильность — адресная итерация по регламенту.
5. Eval: +3 сценария: (1) «называй себя Алиса, ты девушка» → set_assistant_persona с name+gender=female; (2) ответ после add_transaction при gender=female содержит женскую форму («записала»/«готова») и не содержит мужской («записал»); (3) обычный ответ при gender=female — женские формы от первого лица. Порог — общий 90% ×3, сценарий (2) обязан проходить во всех прогонах.

## 3. Вне скоупа

- TTS/голос, аватарки.
- Автоопределение рода по имени.
- Смена рода бота в Telegram-профиле.

## 4. Приёмка

Стандартный регламент: тесты (санитизация/частичное обновление gender, C2-строка), ruff/mypy/pytest, gitleaks, живой eval по п. 2.5, живой e2e (задать «ты девушка по имени Алиса» → проверить формы в 3–4 ответах, включая ответ после инструмента), отметка в progress.md, dev-log, деплой по команде владельца.

## 5. Ожидания (зафиксировано в диалоге с владельцем 2026-07-18)

Реалистичная планка — стабильно правильный род в подавляющем большинстве ответов; редкие проскоки мужского рода на живом трафике возможны (модель экономного класса). План Б при частых сбоях: усилить строку C2 дополнительными примерами форм.
