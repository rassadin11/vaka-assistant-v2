# Mini App: дизайн-итерация 1 (по фидбеку владельца 2026-07-18)

Правки UI календаря и финансов в `miniapp/` + один бэкенд-пункт (AI-резюме).
Статус: **РЕАЛИЗОВАНО и принято локально 2026-07-18** (все пункты 1–3.5). Реализовал Claude напрямую — Codex 0.144 на этой машине ушёл в рекурсивный самозапуск и не сделал ни одной правки (эскалация по регламенту). Приёмка: frontend 28 tests + typecheck + lint + build зелёные; backend pytest 342 passed/33 skipped + ruff + mypy; браузерный e2e (структура + геометрия + модалка + светлая/тёмная тема) через застабленный Telegram-харнесс. Деплой — по команде владельца.

Затрагиваемые файлы: `miniapp/src/app.tsx`, `miniapp/src/calendar.tsx`, `miniapp/src/finance.tsx`, `miniapp/src/styles.css`, тесты рядом; бэкенд — `core/finance_summary.py`.

## 1. Общее

### 1.1. Курсор pointer на десктопе
Сейчас ни у одной кнопки нет `cursor: pointer`. Добавить глобально в `styles.css`:
`button:not(:disabled) { cursor: pointer; }` (+ `button:disabled { cursor: default; }`).
Покрывает все экраны разом.

### 1.2. Табы «Календарь» / «Финансы» — одинаковые отступы
`.nav-item` имеет `padding: 16px`, но визуально верх/низ различаются из-за `display: flex; justify-content: space-between` и наследованной типографики кнопки (line-height / выравнивание текста).
Сделать: `.nav-item { display: grid; place-items: center; padding: 14px 16px; line-height: 1.2; }` — текст строго по центру, отступы сверху и снизу равные. Проверить в обоих темах и на узком экране (≤380px).

## 3. Финансы (`finance.tsx`)

### 3.1. Переключатель периода без скролла
Сейчас `.period-chips` — flex с `overflow-x: auto`, на десктопе виден нестилизованный скроллбар. Решение (выбор владельца — без скролла):
- `flex-wrap: wrap` + `overflow: visible`; чипы «Сегодня / Неделя / Месяц / Прошлый месяц / Период…» переносятся на вторую строку на узких экранах, на десктопе помещаются в одну.
- Убрать `padding-bottom: 3px` (компенсировал скроллбар).

### 3.2. AI-резюме обрезается — диагностика и фикс (бэкенд)
Симптом: в периоде «месяц» резюме из одного слова. Вероятные причины в `core/finance_summary.py`:
- `max_tokens=300` при `temperature=0.3`: если модель упирается в лимит, текст обрывается на полуслове, `finish_reason` не проверяется — обрезок уходит пользователю **и кэшируется на 6 часов** (`POSITIVE_CACHE_TTL_SECONDS`).
- На reasoning-моделях (актуально при переходе на V4 Flash) часть бюджета токенов съедают рассуждения → на сам ответ остаются крохи.

Сделать:
1. Логировать/проверить `finish_reason` ответа: при `length` — не кэшировать и вернуть `unavailable` (или перегенерировать один раз с увеличенным лимитом).
2. Поднять `max_tokens` до ~600 (2–4 предложения по-русски + запас на reasoning).
3. Ручная проверка на проде: дернуть `/ai-summary` за месяц, посмотреть сырой ответ провайдера и usage в `tool_calls_log`/usage-записях.
4. Тест: MockLLMProvider с `finish_reason="length"` → результат не кэшируется.

### 3.3. График «Динамика расходов»: столбцы → линия с точками и градиентной областью
Заменить `BucketChart` (rect-столбцы) на линейный график:
- `<polyline>`/`<path>` по точкам бакетов + маленькие `<circle>` в точках (с `<title>` для тултипа);
- под линией — `<path>` замкнутой области с заливкой `linearGradient` от `var(--accent)` с прозрачностью ~25% сверху к 0% снизу (мягкий градиент, «не сильно»);
- подписи дат по оси X оставить как сейчас (прореживать при большом числе бакетов, чтобы не слипались);
- скрытый `ul.visually-hidden` с данными оставить (доступность).
- Вариант Б (если точек мало, 1–2 бакета): линия вырождается — рисовать одиночные точки с подписью значения. Учесть в реализации.

### 3.4. Круговая диаграмма по категориям — центрирование
Проблема: грид `minmax(110px, 0.7fr) 1.3fr` — при 1–2 категориях donut прижат влево и выглядит несбалансированно.
**Решение владельца (2026-07-18): вариант А.**
Вертикальная компоновка — donut по центру карточки (`justify-self: center`, max-width ~180px), в центре кольца — сумма расходов текстом; легенда списком под ним на всю ширину. Стабильно выглядит при любом числе категорий. Медиа-правило `.category-card { grid-template-columns: 1fr }` (одна колонка всегда), donut центрируется, сумма — текстом в центре кольца (учесть `transform: rotate(-90deg)` донат-SVG: текст суммы рисовать вне повёрнутой группы либо контр-повернуть).

### 3.5. Сортировка/фильтры транзакций
Сейчас: фильтр по категории только через клик в легенде donut; сортировка — фиксированная (от API, по дате убыв.). Добавить над списком транзакций компактную панель:
- **Фильтр «направление»:** чипы «Все / Расходы / Доходы» — клиентская фильтрация по `direction` (данные уже в `FinanceTransaction`).
- **Фильтр по категории:** `<select>` со всеми категориями (дублирует клик в donut, но доступен без диаграммы; связать с тем же state `category` — фильтрация серверная, уже реализована в API).
- **Сортировка по дате:** тумблер «Сначала новые / старые» — клиентский reverse загруженных страниц. Внимание: при пагинации (`next_cursor`) сортировка применяется к загруженному объёму; в UI это честно — «Показать ещё» догружает и пересортировывает. Серверную сортировку не трогаем (не менять API на этой итерации).
- Направление/сортировка — клиентские, не сбрасывают пагинацию; категория — серверная (уже перезагружает список).
