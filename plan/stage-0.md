# Этап 0. Подготовка (2–3 дня)

Часть мастер-плана: [../implementation_plan_v1.md](../implementation_plan_v1.md) — там архитектурные решения, сводный график и регламент передачи задач.

0.1. Документ требований: 5 сценариев (финансы, Google Calendar/Gmail-send, поиск SearXNG, PDF, напоминания), монетизация (подписка ~249₽ через Tribute), политика данных (GDPR-минимум: экспорт, удаление, сроки хранения).
0.2. Репозиторий: monorepo, структура пакетов (gateway/, worker/, core/, tools/, migrations/, infra/), pre-commit (ruff, mypy), pytest.
0.3. CI: lint + tests + сборка образа на каждый push.
0.4. Окружения: dev (локальный docker-compose: postgres, redis, searxng) и prod (сервер).
0.5. Регистрация внешних сущностей — запускается сразу, так как содержит ожидания:
  - Telegram-бот (BotFather), тестовый и боевой.
  - OpenRouter-аккаунт, ключ, лимиты расходов.
  - Google Cloud проект + OAuth consent screen; подача на верификацию scope calendar.events + gmail.send. [Обсудить: точный состав scopes и материалы для верификации — отложенное обсуждение №10]
  - Домен + DNS.
  - Tribute-аккаунт (можно позже, к этапу 7).
0.6. Юридические артефакты: privacy policy — страница публикуется на домене до подачи заявки в Google (требование верификации; хостинг любой, хоть GitHub Pages, сервер не нужен). В тексте раскрыть передачу данных LLM-провайдерам через OpenRouter (только no-training провайдеры — см. «Политика данных LLM» в мастер-плане) и STT-провайдеру для голосовых. Публичная оферта — черновик; финализируется к этапу 7 (Tribute).
