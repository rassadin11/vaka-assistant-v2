# Этап 4. Инструменты (7–14 дней)

Часть мастер-плана: [../implementation_plan_v1.md](../implementation_plan_v1.md) — там архитектурные решения, сводный график и регламент передачи задач.

Содержание полностью в [../tools_registry_v1.md](../tools_registry_v1.md). Порядок внутри этапа:

4.1. Скелет реестра и диспетчера (раздел 3 спецификации) + тесты контракта.
4.2. Финансы (раздел 5) — обкатка контракта без внешних зависимостей.
4.3. Напоминания (6.1–6.3) + процесс-шедулер (тик 60 с, croniter).
4.4. Память (раздел 7): эмбеддинги локальные (решено, обсуждение №4). Мини-бенчмарк bge-m3 vs multilingual-e5-large на русском (20–30 пар «запрос → релевантный факт»), фиксация победителя. Инференс — отдельный внутренний сервис: одна копия модели в RAM, воркеры ходят по HTTP/unix-socket (в каждом воркере модель не поднимать — умножается на число процессов). Автоинжект top-5, дедупликация. Vector-колонки и HNSW уже созданы миграцией v1. Недоступность сервиса эмбеддингов не роняет задачу: блок D (факты) пропускается с записью в лог, remember_fact возвращает retryable-ошибку; алерт — в 6.4.
4.5. SearXNG: развёртывание internal-only + web_search и fetch_page с SSRF-фильтром (раздел 8).
4.6. PDF-пайплайн + search_documents (раздел 9).
4.7. Google: OAuth-флоу (callback, шифрованное хранение в Postgres, фоновый refresh) → list_events, create_calendar_event, send_email с подтверждениями через outbox (раздел 10). Стартовать при готовой верификации из 0.5. [Обсудить: OAuth-флоу целиком]
4.8. Механика подтверждений mutating_external: pending-ключи, inline-кнопки, outbox-доставка (раздел 2 спецификации) — реализуется в составе 4.1, проверяется на 4.7.
4.9. Голосовые сообщения (раздел 11 спецификации): envelope kind=voice → воркер скачивает файл (Bot API) → STTProvider (интерфейс + Mock, реализация по выбранному провайдеру) → транскрипт идёт обычным агентным путём как текст пользователя. Ограничения: длительность ≤ 5 мин (дольше — отказ с сообщением), лимит минут/день — счётчик Redis; стоимость STT пишется в usage (model = 'stt:<provider>', токен-поля NULL). Независим от других пунктов этапа; к закрытой бете обязателен — голосовые пришлют в первый же день. [Обсудить: выбор STT-провайдера — отложенное обсуждение №6]

[DoD] Каждый инструмент: тесты зелёные + ручной e2e через Telegram + запись в tool_calls_log. Голосовое сообщение распознаётся и обрабатывается end-to-end. К бете обязательны: /feedback (пересылка сообщения админу — канал обратной связи беты) и финальные тексты welcome//help из 2.9. Веха: закрытая бета (10–20 пользователей через ручное одобрение из 2.9).
