# Mini App — дизайн-итерация 2

Внеплановая задача (по замечаниям владельца 2026-07-18, после приёмки итерации 1 — см. [miniapp-design-iteration-1.md](miniapp-design-iteration-1.md)). Экраны Preton/Preact: `miniapp/src/`. Реализация — силами Claude (без делегирования: codex на машине нестабилен, разрешение владельца дано явно).

## Замечания и решения

### A. Скелетоны загрузки (вместо текста «Загружаем…»)
- **A1. Календарь.** В состоянии `loading` вместо `CalendarState title="Загружаем календарь…"` показывать скелетон, повторяющий раскладку: тулбар (стрелки + месяц), сетка 7×6 дней, карточка «что на выбранную дату» (заголовок + пара строк).
- **A2. Финансы.** В состоянии `loading` вместо `FinanceState title="Загружаем финансы…"` показывать скелетоны всех блоков: итоги за период, круговая диаграмма, график динамики, список транзакций. Переключатель периода (`PeriodSelector`) остаётся видимым (он вне ветки загрузки).
- Общий класс `.skeleton` с shimmer-анимацией (переиспользует `@keyframes ai-summary-pulse`); при `prefers-reduced-motion` анимация отключается. Скелетоны помечены `aria-busy="true"` и **не** содержат `svg[role="img"]` (чтобы не ломать подсчёты в тестах).

### B. «Дёрганье» экрана при переключении вкладок
- Причина: (1) навигация в потоке над контентом; (2) появление/исчезновение вертикального скроллбара меняет ширину контента (горизонтальный сдвиг) при разной высоте экранов.
- Решение: `scrollbar-gutter: stable` на корне (резервирует место под скроллбар — горизонтальный сдвиг уходит) + скелетоны примерно нужной высоты (меньше вертикального «прыжка» при подстановке контента). Навигация уходит вниз из потока (см. C), верх контента больше не смещается.

### C. Нижняя фиксированная навигация с иконками
- Кнопки «Календарь»/«Финансы» переносятся вниз экрана, `position: fixed`, панель во всю ширину с внутренним контейнером `max-width: 680px` (совпадает с `.shell`), верхняя граница + фон `--surface`, учёт `env(safe-area-inset-bottom)`.
- Каждая кнопка: иконка (24px) сверху + короткая подпись снизу (текст оставляем для однозначности; icon-only отвергнут — календарь и финансы должны различаться без домысла). Активная вкладка — цвет `--primary` + мягкая подложка.
- `.shell` получает нижний `padding-bottom`, равный высоте панели + safe-area, чтобы последний контент (кнопки «Показать ещё» / «+ Напоминание») не перекрывался.
- `.status` (экран авторизации/ошибки) панель НЕ показывает — нижний отступ не добавляется.

### D. Финансы: скрыть «Динамику расходов» для одного дня
- График динамики имеет смысл от 2 точек. Условие рендера меняется `by_bucket.length > 0` → `by_bucket.length > 1`. Это автоматически убирает график для периода «Сегодня» и любого одно-дневного диапазона.

### E. Финансы: стрелка селекта категории у самого края
- `<select>` категории: `appearance: none` + собственный шеврон (декоративный SVG, `currentColor`, `pointer-events: none`), позиционированный `right: 12px`; `padding-right` под шеврон. Тема-безопасно (без data-URI).

### F. Финансы: «Удалить» → иконка корзины
- В карточке транзакции текстовая кнопка «Удалить»/«Удаляем…» заменяется на кнопку-иконку корзины (`aria-label="Удалить транзакцию"`, `aria-busy` во время удаления). Раскладку строки перевести на один ряд: эмодзи | текст | сумма | корзина.
- Тесты, ищущие кнопку по тексту «Удалить», перевести на поиск по `aria-label` (как уже сделано для «Отменить» в календаре). Диалог подтверждения (`window.confirm`) не меняется.

### G. Финансы: крупнее подписи на графике «Динамика расходов»
- Первопричина мелкого текста: `viewBox` растёт с числом точек (`count*44`), а SVG сжимается в ширину контейнера (`width:100%`) — при ~30 днях масштаб ≈0.27, и 9px превращаются в ~2–3px.
- Решение: фиксированный `viewBox` (ширина ~340, высота ~170) независимо от числа точек; точки распределяются внутри фиксированной ширины. Размер шрифта подписей поднять (ось ~11px, значения ~12px) — после масштабирования читаемо. Прореживание подписей оси и показ значений только при ≤2 точках сохраняются.

### H. Финансы: AI-резюме перенести вниз
- Порядок блоков: итоги → круговая → (чип фильтра) → **динамика** → **AI-резюме** → бюджеты → транзакции. `AiSummaryCard` переносится из начала на позицию после `BucketChart`. Когда график скрыт (один день) — AI-резюме просто следует за круговой (деградирует корректно).
