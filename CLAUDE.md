# Проект: Telegram-сервис персональных ассистентов

Multi-tenant сервис: один бот, общий пул воркеров, изоляция пользователей через PostgreSQL RLS. Стек: Python 3.12, FastAPI, aiogram 3.x, asyncio-воркеры, PostgreSQL 16 + pgvector + PgBouncer, Redis ×2, Alembic, Caddy. LLM — DeepSeek через OpenRouter.

## Инварианты кода (проверяются на каждой задаче)

- Состояние только в Postgres/Redis/S3; gateway и воркеры stateless.
- RLS: каждая транзакция под ролью `app` открывается с `SET LOCAL app.user_id`; session-level SET запрещён (PgBouncer transaction mode). Роли: `migrator` (только Alembic), `app` (рабочая), `service` (BYPASSRLS, только кросс-пользовательские процессы).
- `user_id` никогда не приходит из аргументов LLM — только инжект диспетчером из TaskContext.
- Mutating-инструменты идемпотентны при переигровке (реестр §1.7, ключ `idem:{update_id}:{n}`).
- Тесты не ходят в сеть и не тратят токены: MockLLMProvider / MockSTTProvider.
- Секреты только из Infisical (кроме bootstrap.env); `.env` в репозитории запрещён.
- LLM только через интерфейс LLMProvider; OpenRouter с `data_collection: deny` и allowlist провайдеров из конфига.
- Время: в БД `timestamptz` (UTC); для модели — ISO 8601 в таймзоне пользователя; `users.timezone` всегда явная.
- Миграции обратно совместимы с предыдущей версией кода (expand-contract).

## Делегирование внешним агентам

Помимо встроенных сабагентов (Agent tool) доступны внешние CLI-агенты в headless-режиме через Bash. Обвязка:

- скилл `/codex-review` — независимое ревью силами Codex CLI, только чтение, см. `.claude/skills/codex-review/SKILL.md`;
- скилл `/codex-implementation` — реализация по чёткой спеке с правкой файлов, только по явной просьбе пользователя, см. `.claude/skills/codex-implementation/SKILL.md`;
- сабагент `codex-runner` — запуск Codex в изолированном контексте, когда его вывод объёмный и не должен засорять основной диалог.

Когда делегировать (по инициативе Claude — только read-only режимы):

- независимое ревью диффа или плана перед приёмкой — второе мнение другой модели;
- анализ большого объёма файлов, который не нужно тащить в основной контекст, — сабагент `codex-runner`;
- явная просьба пользователя («спроси codex»).

Правила:

- делегат получает самодостаточный промпт со ссылками на плановые документы; наш диалог он не видит;
- режим с правкой файлов (`--full-auto`) — только по явной просьбе пользователя;
- дифф делегата проходит ту же приёмку, что и любой другой (регламент выше); секреты делегатам не передавать;
- вывод делегата — мнение, а не истина: Claude обязан оценить его и явно согласиться или возразить.

Выбор агента (оценки 1–10, больше = лучше; cost отражает фактическую оплату: выше = дешевле):

| агент | cost | intelligence | context | типовое применение |
|-------|------|--------------|---------|--------------------|
| claude fable | 2 | 9 | средний | постановка задач, делегирование работы на другие модели, анализ дальнейшнего плана |
| claude opus 4.8 | 4 | 7 | средний | ревью диффов и планов, сложные правки кода с контекстом проекта |
| claude sonnet 5 | 5 | 5 | средний | сабагенты-исполнители: поиск по кодовой базе, механические правки под присмотром |
| gpt-5.5 | 9 | 8 | средний | массовая реализация по чёткой спеке, независимое ревью, анализ данных |

Как применять таблицу — цель делегирования: максимум качества кода за здоровую цену.

- **Это дефолты, а не лимиты.** Постоянное разрешение на эскалацию: если вывод дешёвой модели не дотягивает до планки — перезапустить или переделать более умной без дополнительного вопроса. Судить по результату, а не по ценнику: эскалация стоит дешевле, чем принять посредственную работу и потом её переделывать.
- **Cost — только tie-breaker.** Цена решает при примерно равном качестве. Для всего, что идёт в приёмку: intelligence > cost.
- **Массовая механика по чёткой спеке** (реализация по stage-N.md с DoD, анализ данных, прочёс файлов/логов): gpt-5.5 — при cost 9 он практически бесплатен, а int 8 достаточно. Спека делегату — плановые документы; его дифф всегда проходит приёмку силами claude (регламент и инварианты gpt не знает).
- **Ревью планов и реализаций**: fable или opus 4.8; gpt-5.5 — опционально как дополнительная независимая перспектива. На ревью не экономить: пропущенный баг дороже токенов.
- **Оркестрация, декомпозиция, архитектурные решения** — fable (основной агент). Он самый дорогой, поэтому его контекст беречь: объёмные выводы уводить в сабагентов, а не читать в основном диалоге.
- **Не опускаться ниже sonnet 5.** Модель ниже планки даёт мнимую экономию — переделка съедает выгоду.
- **Тексты для пользователей бота и API-контракты** — не отдавать самой дешёвой модели: формулировки и интерфейсы проверяет fable/opus.
- **Механика запуска**: gpt-5.5 доступен только через Codex CLI — скиллы `/codex-review` (ревью, read-only) и `/codex-implementation` (реализация по спеке, с правками); для задач, не покрытых скиллами (исследование, анализ данных), — `codex exec --sandbox read-only --skip-git-repo-check` с самодостаточным промптом; объёмный вывод — через сабагента `codex-runner`. Claude-модели — через Agent tool, параметр `model` (sonnet / opus; fable — основной диалог).

gpt-5.5 внутри сабагентов и Workflow (параметр `model` принимает только Claude-модели, поэтому нужна обёртка):

- обёртка — тонкий Claude-агент (`model: 'sonnet'`, `effort: 'low'`), который составляет самодостаточный промпт для codex, запускает `codex exec` через Bash и возвращает отчёт. Готовая обёртка — сабагент `codex-runner`; в Workflow вызывать через `agentType: 'codex-runner'`, для структурированного ответа добавлять `schema`;
- всегда маркировать такие вызовы префиксом в label: `{label: 'gpt-5.5:review-auth'}` — UI показывает Claude-модель обёртки, и только label говорит, что реальный исполнитель gpt-5.5;
- запуски codex могут превышать таймаут Bash (по умолчанию 2 мин): передавать явный `timeout` (до 10 мин) или запускать в фоне и ждать файла-отчёта;
- параллельные codex-агенты с правкой файлов — только с `isolation: 'worktree'`, чтобы правки не сталкивались в общем чекауте. Требует git-репозиторий: до `git init` параллельные запуски с правками запрещены;
- бюджеты токенов Workflow считают только токены Claude: работа codex не видна в `budget.spent()` и практически бесплатна — учитывать при оценке стоимости фан-аута.
